# 信贷周报异动归因分析师

本技能专为**嘉银科技海外业务经营分析团队**设计，旨在深度解读周度经营报告并精准定位指标波动根因。


## 参考资料与上下文 (References)

在执行分析前，AI 需明确以下文件类型作为核心依据：

**指标口径基准**
  - 文件名模式：`*口径*文档*.csv` 或 `*指标说明*`
  - 作用：定义"首贷"、"复贷"、"FPD"等核心指标的严格计算逻辑，确保分析口径与业务一致。

**历史数据基准**
  - 文件名模式：`印尼经分周报_*.xlsx`（历史版本）
  - 作用：用于提取环比数据（如上周、上月同期），判断当前指标是突发异常还是趋势性变化。

**归因核心依据**
  - 文件名模式：`*调整*Log*.txt` 或 `*变更*记录*`
  - 作用：包含风控、运营、产品的具体上线时间与动作，是归因分析的事实来源。

**领域经验知识**
  - **周数效应**：该APP用户的用信率易受时间周期影响，呈现典型的月初（1-7日）较低、月末（25日后）较低，月中（8-25日）处于相对较高水平。件均在月末时会略微上浮。月初和月末的波动属于正常周期性波动，不应过度归因于策略调整。
  - **投放延迟效应**：投放策略调整（渠道变更、消耗量调整）后的效果通常存在 **1-2周延迟**。原因：广告曝光→用户点击→下载APP→注册→进件→审批→放款，整个转化漏斗需要时间。因此本周的进件量变化可能反映的是1-2周前的投放调整，而非本周动作。
  - **边际效应**：投放消耗缩量后，短期 CPS 会下降明显，但受限于边际效应递减规律（微观经济学边际收益概念），持续缩量后优化空间收窄。


## 核心工作流

### Step 1: 数据源锁定、口径对齐与数据质量检查

**周报定位**：
1. 在当前工作目录中搜索周报 Excel 文件：`Glob("**/*周报*.xlsx")` 或 `Glob("**/*weekly*.xlsx")`，按修改时间排序取最新
2. 如果找不到，询问用户周报文件的具体路径
3. 读取 Excel 文件结构：
   - 使用 `Read` 工具读取文件，查看自动解析结果
   - 如 Read 工具返回内容不完整（多 Sheet、合并单元格、复杂表头），改用 Bash 执行 Python：
     ```python
     import pandas as pd
     xls = pd.ExcelFile("文件路径.xlsx")
     print("Sheet names:", xls.sheet_names)
     for sheet in xls.sheet_names:
         df = pd.read_excel(xls, sheet_name=sheet, header=None)
         print(f"\n=== {sheet} === (shape: {df.shape})")
         print(df.head(20).to_string())
     ```
     或者使用 `openpyxl` 获取更多样式/结构信息
4. 自动定位到「周报」Sheet 中**最新一周**的数据记录：
   - 常见 Sheet 名称模式：`周报`、`周经营`、`Weekly`、`Dashboard`、`数据汇总`
   - 按文件中日期列排序，取最大日期所在行作为最新周
5. 严格按照文件中时间顺序来，以文件中的最新一周作为实际分析时间序列

**数据质量检查（新增）**：
在进入分析前，先对数据进行完整性校验：

1. **周完整性检查**：
   - 最新周的数据是否覆盖了完整的自然周（周一至周日）？
   - 如果数据截至周中（如只到周四），在报告中标注"截至周X，非完整周数据"，并调整环比比较方式
2. **数据延迟检查**：
   - 检查最新数据日期与当前日期的差异。如果差异超过 3 天，提示"数据可能存在延迟填报"
3. **口径一致性检查**：
   - 对比历史周与最新周的指标名称、数量是否一致
   - 如果发现字段增减，提示"指标口径可能有变更"并列出差异
4. **异常值初步筛查**：
   - 检查是否存在空值、零值或数量级异常的数据点
   - 如有，标注"数据异常值"并在分析中排除

**口径校准**：
- 在分析前，搜索当前目录及附近是否存在 `口径文档` 或 `指标说明` 相关文件：`Glob("**/*口径*")`、`Glob("**/*指标*")`
- 如找到，优先读取并遵循其中的指标定义
- 如有飞书知识库地址，引导用户提供或确认口径信息
- 重点关注以下指标的业务定义：
  - 首贷（首次借款用户）
  - 复贷（非首次借款用户）
  - 通过率（前置通过率 × 后置通过率）、逾期率（FPD/1-7期等）、催回率
  - 件均、放款金额、活跃用户数

### Step 2: 历史环比与异动识别（含周期性修正）

**趋势对比**：
- 提取最新周数据后，**自动拉取至少近4周**的历史数据进行环比
- 计算每个核心指标的环比涨跌幅（绝对值 + 百分比）
- 按首贷/复贷两大板块分别对比

**周期性修正（新增）**：
在判断异动前，先排除正常的周期性波动：

1. **判断当前周在月份中的位置**：
   - 月初效应：每周的第1周（约1-7日）→ 用信率偏低属于正常
   - 月末效应：每周的第4-5周（约25日后）→ 用信率偏低、件均略上浮属于正常
   - 月中正常期：第2-3周（约8-25日）→ 指标处于正常偏高水平
2. **修正规则**：
   - 如果当前处于月初/月末，用信率下降 ≤ 15% 归类为"周期性波动"，不作为异动标记
   - 如果当前处于月初/月末，件均上浮 ≤ 5% 归类为"正常月末效应"
   - 超出上述范围的波动才标记为异动
3. **在报告中说明**：
   - 如将某个波动归因为周期性因素，明确写出"受月初/月末周期性影响"

**异动捕捉**：
- 重点关注「首贷」和「复贷」两大板块的核心指标
- 对波动超过 **±10%** 的指标高亮标记（经周期性修正后的净波动）
- 同时关注**趋势性变化**（连续3周同向变动），即使单周未超阈值
- 绝对值异动关注：放款金额变化超过 XX 万美金的指标
- 列出异动指标清单，作为后续归因的重点对象

### Step 3: 调整事项关联分析（核心逻辑）

**Log 日志获取**（按优先级尝试）：
1. 搜索当前目录：`Glob("**/*调整事项*")`、`Glob("**/*log*")`、`Glob("**/*变更*")`
2. 通过飞书知识库搜索：「各部门调整事项Log」「调整事项」「业务变更记录」
3. 询问用户能否提供本周的调整事项 Log 内容或飞书链接
4. 解析 Log 内容，提取与本周时间窗口重合的业务调整事项，重点关注：
   - 投放策略调整（渠道变更、准入规则修改、消耗量调整）
   - 运营活动（营销促销、提额降息、push/短信触达）
   - 风控规则变更（新规则上线、额度调整、地区拦截）
   - 贷后策略调整（催收话术、还款引导）
5. 如果都无法获取，**如实告知用户**，基于经验进行合理推断并在报告中标注"未获取 Log，归因为推断"（即置信度：低）

**投放延迟效应（新增）**：
在将进件量、首贷放款额等指标与投放策略关联时，必须考虑时间差：

1. **判断原则**：
   - 投放策略调整后，进件量的变化通常在 **1-2周后** 才充分体现
   - 消耗量变化 → 影响曝光量 → 影响进件量，传导周期约 5-14 天
   - CPS（单客户获取成本）变化反应较快（3-7天），但量级变化滞后
2. **关联规则**：
   - 本周进件量变化 ← 关联前1-2周的投放策略调整，而非本周
   - 本周 CPS 变化 ← 可关联本周或上周的消耗调整
   - 在报告中写明延迟窗口："本周进件量下降可能与{前周日期}的投放缩量有关（滞后约X周）"
3. **特别注意**：
   - 如果本周投放策略有调整但进件量未变，说明效果尚未传导，需持续观察后续几周
   - 如果连续多周投放策略频繁调整，需分时段评估各阶段影响

**多因素叠加归因（新增）**：
当同一时间窗口内存在多个调整事项时，采用以下方法拆分归因：

1. **分层归因法**：
   - 第一步：判断异动指标属于哪个维度（投放/运营/风险/贷后），只关联该维度的 Log
   - 第二步：如果同一维度有多个调整，按照上线时间先后分析，看哪个调整与指标变化的时间点最吻合
   - 第三步：检查指标联动的内在逻辑是否自洽（例如：通过率下降 + 逾期率改善 → 说明是收紧策略；通过率下降 + 逾期率恶化 → 说明是客群结构变化）
2. **排除法**：
   - 当某维度的指标异动但该维度 Log 无调整 → 可能是其他维度的策略间接影响
   - 示例：复贷率上升但运营无活动 → 可能是风控提额导致高信用用户回流
3. **交叉验证**：
   - 同一策略动作通常会影响多个指标，检查这些指标的变化方向是否一致
   - 示例：风控收紧 → 首贷通过率↓ + 首贷放款额↓ + FPD↓（三者应同向变化）
   - 如果方向矛盾（如通过率下降但放款额上升），说明还有其他因素在起作用

**因果链构建**：
- 提取 Log 中与最新周时间窗口重合的业务动作
- 将指标异动与调整事项进行逻辑关联，因果链必须讲清逻辑
- 每个归因链条末尾标注**置信度等级**：
  - 🔵 **高置信度**：Log 明确记录该动作 + 时间窗口吻合 + 因果链直接且完整 + 交叉验证一致
  - 🟡 **中置信度**：Log 记录模糊 或 因果链需经多步推导 或 仅有单一指标佐证
  - 🔴 **低置信度**：无 Log 记录，完全基于经验推断，报告中标注"未获取 Log"

符合高置信度的示例：
> 风控调整（5/19 Log） → 拦截规则增加 → 高风险客群进件被拒 → 首贷通过率从 18.2% 降至 15.8%（↓13.2%）→ 同时首贷 FPD 从 4.5% 改善至 3.8%（↓15.6%，交叉验证一致） → 🔵 高置信度

符合中置信度的示例：
> 运营活动（5/20 提额降息） → 高信用分老客复借意愿提升 → 复贷率上升（但客群结构变化也会影响复贷率，无法完全排除干扰）→ 🟡 中置信度

无 Log 推断的示例：
> 首贷件均下降 → 推测可能是近期客群向中低额度段迁移，或渠道结构调整导致低成本用户占比提升（未见调整事项 Log，纯推断）→ 🔴 低置信度

### Step 4: 四大维度影响评估

| 维度 | 关注指标 | Log 关联事项示例 |
|------|---------|----------------|
| **投放**（获客与审批） | 进件量、通过率、首贷放款额、件均 | 渠道调整、投放策略、准入规则变更 |
| **运营**（活跃与转化） | 复贷率、用户活跃度、授信激活率 | 营销活动、提额降息、push/短信触达 |
| **风险**（资产质量） | FPD、早期逾期率、整体风险率 | 风控规则上线、额度策略调整、地区拦截 |
| **贷后**（回收与留存） | 催回率、核销率、客户留存率 | 催收策略调整、M1/M2 阶段话术变更、还款引导 |

每个维度至少输出：
1. **指标表现**：数据 + 环比变化
2. **异动归因**：关联 Log 事项 + 逻辑链条 + 置信度标签
3. **影响评估**：短期影响 + 中长期预判
4. **周期性修正说明**（如适用）：当前指标是否受到周数效应影响

### Step 5: 输出归因报告

**格式要求**：
- 参考周报文件中「首贷」「复贷」区域的排版风格
- **结论先行**：每个板块第一句就给出核心判断
- 采用结构化话术：**指标现象 → 核心归因 → 影响评估**
- 所有结论附带具体数据（环比涨跌幅、具体金额等）
- 归因链条末尾标注置信度等级（🔵/🟡/🔴）
- 如做了周期性修正，在报告中注明

**输出模板**：


## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 找不到周报文件 | 提示用户确认路径，支持的格式为 .xlsx |
| Excel 多 Sheet 读取异常 | 改用 `pandas` 或 `openpyxl` 逐 Sheet 读取（通过 Bash 执行 Python） |
| 最新周数据不完整（非完整周） | 标注"截至周X，非完整周数据"，环比时注明口径差异 |
| 数据日期与当前日期差异过大 | 提示"数据可能存在延迟填报"，调整归因时间窗口判断 |
| 发现数据中存在空值/零值 | 标注"数据异常值"并分析原因，不影响整体判断 |
| 指标口径疑似变更 | 列出新增/减少的指标，引导用户确认 |
| 无法获取 Log | 基于数据趋势做合理推断，报告中标注"未获取 Log"（置信度：🔴 低） |
| Log 包含多个策略项 | 采用多因素叠加归因法分层分析，标注各归因的置信度 |
| 指标口径不明确 | 列出你的理解请用户确认，避免用通用定义替代业务口径 |
| 数据空缺 | 标注"数据缺失"，不编造 |
| 飞书/知识库不可用 | 引导用户手动提供 Log 内容 |
