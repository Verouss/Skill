# 信贷周报异动归因分析师

本技能专为**嘉银科技海外业务经营分析团队**设计，深度解读周度经营报告并精准定位指标波动根因。


## 参考资料与上下文

**指标口径基准** — 文件名模式 `*口径*文档*.csv` 或 `*指标说明*`。定义首贷、复贷、FPD 等核心指标的严格计算逻辑。

**历史数据基准** — 文件名模式 `印尼经分周报_*.xlsx`。用于提取环比数据，判断指标是突发异常还是趋势性变化。

**归因核心依据** — 文件名模式 `*调整*Log*.txt`。包含风控、运营、产品的具体上线时间与动作，是归因的事实来源。

**分析框架引用**（当涉及以下专项分析时，必须读取对应的参考文件并遵循其中的方法）：
- **CPS 异动分析** → 读取 `references/frameword-cps.md`，按流程拆解→渠道结构分析→贡献度估算三步进行
- **用信率异动分析** → 读取 `references/framework-confirmrate.md`，按客群结构→mix效应→贡献度估算进行

**领域经验知识**：
- **周数效应**：用信率呈月初（1-7日）较低、月末（25日后）较低，月中（8-25日）较高。件均月末略上浮。月初/月末波动 ≤15% 属于正常周期性波动，不应过度归因于策略。
- **投放延迟效应**：投放调整后的效果通常有 **1-2周延迟**（广告曝光→点击→下载→注册→进件→审批→放款）。本周进件量变化可能反映1-2周前的投放调整，而非本周动作。
- **边际效应**：投放消耗缩量后短期 CPS 下降明显，但受边际效应递减规律约束，持续缩量后优化空间收窄。


## 核心工作流

### Step 1: 数据源定位、口径校准与数据质量检查

**周报定位**：
1. 搜索周报文件：`Glob("**/*周报*.xlsx")` 或 `Glob("**/*weekly*.xlsx")`，取最新的
2. 找不到则询问用户路径
3. 读取文件结构：Read 工具不完整时，用 Bash 执行 Python（pandas/openpyxl）逐 Sheet 读取
4. 定位到最新一周数据（按日期列排序取最大日期）

**数据质量检查** — 发现问题后仍要继续分析，不能卡住：

1. **周完整性**：最新周是否完整自然周？若不完整（如只到周四），标注"非完整周数据"，环比改为日均化比较
2. **数据延迟**：最新数据日期与当前日期差异 > 3天 → 提示"可能存在延迟填报"
3. **口径一致性**：对比历史周与最新周的指标名称数量是否一致，如有增减说明差异
4. **异常值筛查**：检查空值、零值或数量级异常。如有空值 → 标注"数据异常值"并跳过该指标，**不要编造数值**

**口径校准**：搜索 `Glob("**/*口径*")` 和 `Glob("**/*指标*")`，找到后优先遵循其中的指标定义。

### Step 2: 环比分析与异动识别（含周期性修正）

**趋势对比**：拉取至少近 4 周历史数据，计算核心指标环比涨跌幅，按首贷/复贷两大板块分别对比。

**周期性修正**（判断当前周在月中的位置）：
| 位置 | 日期范围 | 正常现象 | 修正规则 |
|------|---------|---------|---------|
| 月初 | 第1周(约1-7日) | 用信率偏低 | 下降≤15%归为周期性波动 |
| 月中 | 第2-3周(约8-25日) | 指标正常偏高 | 无需修正 |
| 月末 | 第4-5周(约25日后) | 用信率偏低、件均上浮 | 用信率↓≤15%归为周期性、件均↑≤5%归为月末效应 |

归因为周期性因素的波动，在报告中明确写"受月初/月末周期性影响"，不标记为异动。

**异动捕捉**：
- 经周期性修正后的净波动超过 **±10%** 的指标高亮标记
- **趋势性异动**：连续3周同向变动即使单周未超阈值也要关注，说明策略效果在持续释放
- 绝对值异动：放款金额变化超过 XX 万美金的指标
- 列出异动指标清单作为后续归因重点对象

### Step 3: Log 关联与归因分析（核心逻辑）

**Log 获取**（按优先级）：
1. 搜索飞书知识库：「各部门调整事项Log」「调整事项」「业务变更记录」
2. 询问用户能否提供 Log 内容或飞书链接
3. 解析 Log，提取与本周时间窗口重合的四类调整：
   - **投放**：渠道变更、准入规则修改、消耗量调整
   - **运营**：营销活动、提额降息、push/短信触达
   - **风控**：新规则上线、额度调整、地区拦截
   - **贷后**：催收策略调整、话术变更
4. 完全无法获取时 → 如实告知用户，基于数据趋势做合理推断，报告标注"未获取 Log"

**投放延迟效应**（关联投放与进件指标时的核心约束）：
- 投放调整 → 进件量变化约 **1-2周后** 才充分体现（曝光→下载→注册→进件，传导5-14天）
- 本周进件量 ← 关联前1-2周投放调整，而非本周
- CPS 变化 ← 可关联本周或上周调整（反应较快）
- 报告写明延迟窗口："本周进件量下降可能与{前周日期}的投放缩量有关（滞后约X周）"

**多因素叠加归因**（同一窗口多个调整时的拆分方法）：

分层归因 + 排除法 + 交叉验证，三者配合使用：
1. **分层归因**：判断异动指标属于哪个维度（投放/运营/风险/贷后），只关联该维度 Log
2. **交叉验证**：同一策略影响多个指标，检查变化方向是否一致
   - 风控收紧 → 首贷通过率↓ + FPD↓（一致）；通过率↓ + FPD↑ → 是客群下沉而非收紧
3. **排除法**：某维度指标异动但该维度无 Log → 可能是其他维度间接影响（如复贷率↑但无运营活动 → 可能是风控提额导致高信用回流）

**因果链构建与置信度等级**：
每个归因链条末尾标注等级，注意：置信度不是根据推导步骤数量，而是根据证据的确定性：

| 等级 | 条件 | 示例 |
|------|------|------|
| 🔵**高** | Log明确记录 + 时间吻合 + 因果链直接 + 交叉验证一致 | 风控(5/19)拦截规则→通过率↓+FPD↓ |
| 🟡**中** | Log有记录但模糊，或因果链需多步推导，或仅有单一指标佐证 | 运营活动→复贷率↑（无法排除首贷分母效应） |
| 🔴**低** | 无Log，完全基于经验推断，标注"未获取 Log" | 件均↓→推测渠道结构变化（纯推断） |

### Step 4: 四大维度评估

评估时参照以下维度框架，每个维度至少输出">指标表现 + 异动归因 + 影响评估"三项：

| 维度 | 关注指标 | Log 关联事项 |
|------|---------|-------------|
| **投放** | 进件量、通过率、首贷放款额、件均 | 渠道调整、投放策略、准入规则 |
| **运营** | 复贷率、用户活跃度、授信激活率 | 营销活动、提额降息、push/短信 |
| **风险** | FPD、早期逾期率 | 风控规则上线、额度调整、区域拦截 |
| **贷后** | 催回率、核销率、客户留存率 | 催收策略调整、话术变更 |

### Step 5: 输出归因报告

**格式要求**：
- **结论先行**：每个板块第一句给出核心判断（"首贷放款额环比下降12%"）
- **结构**：指标表现 → 异动归因(置信度) → 影响评估
- **所有结论附带具体数据**（环比涨跌幅、金额等）
- **置信度标签**标注在每个归因链末尾
- 周期性修正的指标在报告中注明

**核心原则**：报告不要过长。聚焦异动指标，正常指标一句带过即可。输出直接到对话框，不生成额外文件。

**输出模板**：

```markdown
# 第X周经营数据异动归因报告
> 数据期间：YYYY-MM-DD ~ YYYY-MM-DD | 完整性：{完整周/截至周X非完整周} | 延迟：{正常/延迟X天} | Log：{已获取/未获取}

## 1. 本周核心概览
{2-3句话概括：最大异动、核心归因}

**周期性修正**：{当前周位置，哪些波动归因为周期性因素}

## 2. 首贷业务
**指标**：放款额 XX万(环比↓X%)，通过率 X%(↓Xpp)，FPD X%(↓Xpp改善)
**归因**：[🔵/🟡/🔴] {策略动作→影响路径→交叉验证}

## 3. 复贷业务
**指标**：复贷率 X%(↑Xpp)，放款额 XX万(↑X%)

**影响**：{正面/负面+可持续性}

## 4. 风险监控
**指标**：FPD X%(±X%)，催回率 X%(±X%)

| {复贷率上升原因} | 🟡中 | Log+推测链条较长 |

```

**格式说明**：模板是参考结构，可根据实际情况灵活调整板块顺序和内容详略。没有异动的维度可合并或省略。


## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 找不到周报文件 | 提示用户确认路径 |
| Excel 读取异常 | 用 pandas/openpyxl 逐 Sheet 读取 |
| 最新周数据不完整 | 标注"截至周X非完整周"，环比改为日均化 |
| 数据延迟过大(>3天) | 提示"可能存在延迟填报"，调整归因时间窗口 |
| 数据有空值/零值 | 标注"数据异常值"并跳过该指标，不编造 |
| 指标口径疑似变更 | 列出差异引导用户确认 |
| 无法获取 Log | 基于数据做合理推断，标注"未获取 Log(🔴低)" |
| 多项策略叠加 | 分层归因+排除法+交叉验证配合使用 |
| 飞书不可用 | 引导用户手动提供 Log 内容 |

不确定时，说"我没有足够的信息来自信地评估这一点。"
